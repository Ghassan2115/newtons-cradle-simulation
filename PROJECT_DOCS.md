# توثيق مشروع بندول نيوتن ثلاثي الأبعاد

## نظرة عامة

محاكاة علمية تفاعلية عالية الدقة لبندول نيوتن (Newton's Cradle) تعمل في المتصفح مباشرة. المشروع يجمع بين محرك فيزياء دقيق ومحرك رسوميات ثلاثي الأبعاد مع واجهة مستخدم تفاعلية كاملة.

---

- **حالات مسبقة الإعداد**: 6 أوضاع جاهزة (أرجحة كرة واحدة، كرتين، كتل متفاوتة، إلخ)
- **عدد الكرات**: من 2 إلى 8 كرات

- **معامل الاسترداد (e)**: من 0.5 إلى 1.0
- **معامل التخميد**: من 0.0 إلى 0.2

- **نظام ألوان الكرات**: كروم / طاقة / سرعة / قوس قزح

#### شريط الحالة (أسفل الشاشة):
- الوقت التراكمي للمحاكاة
- تردد الحساب الفيزيائي (1000 هرتز)
- الخطأ النسبي للطاقة (مؤشر الاستقرار العددي)
- معدل الإطارات الحقيقي (FPS)

### 2. `src/main.ts` — نقطة الدخول وحلقة المحاكاة

هذا هو الملف المركزي الذي يربط جميع المكونات معاً.

#### المتغيرات العامة:
| المتغير | الوصف |
|---------|-------|
| `engine` | كائن محرك الفيزياء |
| `renderer` | كائن المصيّر ثلاثي الأبعاد |
| `dashboard` | كائن لوحة البيانات والرسوم البيانية |
| `soundManager` | كائن مدير الصوت |
| `isPlaying` | هل المحاكاة تعمل أم متوقفة |
| `speedFactor` | مضاعف السرعة البصرية (0.05x - 2.0x) |
| `physicsDt` | الخطوة الزمنية الثابتة = 0.001 ثانية (1000 هرتز) |
| `accumulator` | المجمّع الزمني للخطوات الفيزيائية |

#### الدوال الرئيسية:

**`createConfig()`**: تُنشئ كائن `CradleConfig` يحتوي على كافة الخصائص الفيزيائية. تحسب نقاط التعليق (pivots) تلقائياً بحيث تكون الكرات متلامسة في وضع السكون.

**`initSimulation()`**: تُهيئ المحاكاة لأول مرة: إنشاء المصيّر، لوحة التحكم، مدير الصوت، ربط الأحداث، ثم بدء الحلقة.

**`simulationLoop()`**: حلقة المحاكاة الرئيسية التي تعمل كل إطار:
1. حساب delta time (الفرق الزمني بين الإطارات)
2. تقييد delta إلى 0.1 ثانية كحد أقصى
3. حفظ الحالة السابقة يدوياً (للاستكمال الخطي)
4. تنفيذ خطوات الفيزياء بحد أقصى 16 خطوة لكل إطار
5. تحديث الرسوميات مع الاستكمال الخطي (interpolation)
6. تحديث شريط الحالة
7. تحديث الرسم البياني كل 6 إطارات

**`loadPreset()`**: تُحمّل إعداداً مسبقاً. تدعم 6 أوضاع مختلفة وتُعيد بناء المحرك والمشهد بالكامل.

**`resetSimulation()`**: تُعيد المحاكاة للحالة الابتدائية مع الحفاظ على الإعدادات الحالية.

**`bindUiEvents()`**: تربط جميع أحداث عناصر HTML بالدوال المناسبة.

### 4. `src/physics/Engine.ts` — محرك الفيزياء

القلب الحسابي للمشروع. يُنفذ ثلاث عمليات في كل خطوة:

#### أ) حساب التسارع الزاوي:
```
θ'' = -(g/L) × sin(θ) - (b/m) × θ'
```
حيث:
- `g` = الجاذبية، `L` = طول الخيط
- `b` = معامل التخميد، `m` = الكتلة
- `θ'` = السرعة الزاوية

#### ب) التكامل العددي:
يستخدم المكامل المختار (RK4 أو Verlet) لتحديث الزوايا والسرعات.

#### ج) حل التصادمات (Iterative Impulse Resolution):
- يحسب المسافة الفعلية بين كل زوج من الكرات المتجاورة
- إذا وُجد تداخل (overlap > 0) مع حركة تقاربية:
  ```
  vA⁺ = (mA·uA + mB·uB - mB·e·(uA-uB)) / (mA+mB)
  vB⁺ = (mA·uA + mB·uB + mA·e·(uA-uB)) / (mA+mB)
  ```
- يُكرر حتى 30 مرة حتى تُحل جميع التصادمات المتسلسلة

#### حساب الطاقة:
- **كامنة**: `Ep = Σ(m × g × L × (1 - cos(θ)))`
- **حركية**: `Ek = Σ(0.5 × m × (L × ω)²)`
- **خطأ نسبي**: `|Et - E₀| / E₀`

### 5. `src/physics/integrators/` — المكاملات العددية

#### `RK4.ts` — رونج-كوتا من الرتبة الرابعة:
أعلى دقة (خطأ O(dt⁵) لكل خطوة). يحسب 4 تقديرات وسيطة (k1-k4) ويجمعها بأوزان:
```
θ(t+dt) = θ(t) + (dt/6)(k1 + 2k2 + 2k3 + k4)
```

#### `VelocityVerlet.ts` — فيرليه للسرعة:
أسرع قليلاً ويحافظ على الطاقة بشكل ممتاز:
1. تحديث الموضع: `θ₁ = θ₀ + ω₀·dt + 0.5·α₀·dt²`
2. تنبؤ السرعة: `ωpred = ω₀ + α₀·dt`
3. حساب التسارع الجديد: `α₁ = f(θ₁, ωpred)`
4. تصحيح السرعة: `ω₁ = ω₀ + 0.5·(α₀ + α₁)·dt`

### 6. `src/renderer/CradleRenderer.ts` — المصيّر ثلاثي الأبعاد

يستخدم Three.js لرسم المشهد ثلاثي الأبعاد:

#### مكونات المشهد:
- **الكاميرا**: PerspectiveCamera بزاوية 45° مع OrbitControls
- **الإضاءة**: ضوء محيطي + ضوء اتجاهي (مع ظلال) + ضوء نقطي ملون
- **القاعدة**: صندوق بلون خشبي (Walnut Wood)
- **الهيكل المعدني**: 4 أعمدة رأسية + 4 قضبان أفقية (Chrome)
- **الكرات**: كرات معدنية بخامة `MeshStandardMaterial`
- **الخيوط**: خطوط V-Shape (خيطين لكل كرة)
- **الأسهم**: سهم السرعة (أخضر) وسهم التسارع (أحمر)
- **المسارات**: خطوط تتبع حركة الكرة (60 نقطة)

#### أنظمة الألوان الأربعة:
| النظام | الوصف |
|--------|-------|
| `chrome` | لون رمادي معدني عاكس ثابت |
| `energy` | يتغير بين سماوي (طاقة كامنة) ووردي (طاقة حركية) |
| `velocity` | يتغير من كحلي (سكون) إلى ناري (حركة سريعة) |
| `rainbow` | ألوان الطيف الثابتة لكل كرة |

#### الاستكمال الخطي (Interpolation):
يخلط بين الحالة السابقة والحالة الحالية بنسبة `alpha` لتنعيم الحركة ومنع التقطيع البصري.

## دليل التعديلات الشائعة

### تغيير حجم الكرات يدوياً من الكود

في ملف `src/main.ts`، دالة `createConfig()`:
```typescript
const finalRadii = radii || new Array<number>(ballCount).fill(0.015); // 1.5cm
```
غيّر القيمة `0.015` إلى الحجم المطلوب بالمتر. مثلاً `0.025` لكرات بقطر 5 سم.

> **ملاحظة**: عند تغيير حجم الكرات يدوياً من الواجهة (قسم "تخصيص الكرات الفردية")، يتم حساب نصف القطر تلقائياً من الكتلة باستخدام كثافة الفولاذ (7850 كغ/م³):
> ```
> radius = ∛(3 × mass / (4π × 7850))
> ```

### تغيير عدد الكرات الافتراضي

في `index.html`، ابحث عن:
```html
<input type="range" id="ball-count-range" min="2" max="8" step="1" value="5" />
```
غيّر `value="5"` و `max="8"` حسب الرغبة.

### تغيير السرعة الافتراضية

في `src/main.ts`:
```typescript
let speedFactor = 1.0;
```
غيّر القيمة. مثلاً `0.5` لنصف السرعة أو `2.0` للضعف.

### تغيير لون الخلفية

في `src/renderer/CradleRenderer.ts`، دالة `initScene()`:
```typescript
this.scene.background = new THREE.Color(0xfaf9f0);
this.scene.fog = new THREE.FogExp2(0xfaf9f0, 0.15);
```
غيّر `0xfaf9f0` إلى اللون المطلوب (مثلاً `0x1a1a2e` لخلفية داكنة).

وفي `src/style.css`:
```css
--bg-color: #faf9f0;
```

### إضافة حالة مسبقة جديدة (Preset)

1. في `index.html`، أضف خياراً جديداً في `<select id="preset-select">`:
```html

```

2. في `src/main.ts`، أضف حالة جديدة في `switch` داخل `loadPreset()`:
```typescript
case 'my-preset':
  config = createConfig(ballCount);
  initialAngles = new Array<number>(ballCount).fill(0);
  initialAngles[0] = -0.5; // زاوية مخصصة
  break;
```

### تغيير دقة الفيزياء

في `src/main.ts`:
```typescript
let physicsDt = 0.001; // 1000 Hz
```
- تقليل القيمة (مثلاً `0.0005` = 2000 Hz) يزيد الدقة لكن يستهلك أداءً أكثر
- زيادتها (مثلاً `0.002` = 500 Hz) تقلل الأداء لكن قد تفقد الدقة

### تغيير طول الخيوط

في `src/main.ts`، دالة `createConfig()`:
```typescript
const finalLengths = lengths || new Array<number>(ballCount).fill(0.25); // 25cm
```
غيّر `0.25` إلى الطول المطلوب بالمتر.

### تعطيل الظلال لأداء أفضل

في `src/renderer/CradleRenderer.ts`:
```typescript
this.renderer.shadowMap.enabled = false;
```
وأزل `castShadow = true` و `receiveShadow = true` من جميع الكائنات.

### تغيير صوت التصادم

في `src/ui/SoundManager.ts`، دالة `playClick()`:
- تردد الصوت الأساسي: `osc1.frequency.setValueAtTime(2000, now)` — غيّر 2000 لتردد مختلف
- مدة الصوت: `osc1.stop(now + 0.045)` — غيّر 0.045 لمدة أطول أو أقصر
- شدة الصوت: `const volume = Math.min(0.8, ...)` — غيّر 0.8 للحد الأقصى
